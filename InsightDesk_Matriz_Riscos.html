# Risco Probabilidade Impacto Estratégia de mitigação Responsável
1 Baixa qualidade dos dados de entrada (tickets mal estruturados) Alta Alto Critérios mínimos de qualidade no contrato de onboarding + etapa de limpeza no pipeline de ingestão Tech Lead
2 Alucinação da IA (planos de ação incorretos ou inadequados) Média Alto Revisão humana obrigatória para planos de alto impacto + sistema de feedback contínuo para retreinamento Product Manager
3 Baixa adoção pelos usuários finais (resistência à mudança) Média Alto Onboarding estruturado com treinamento + champion interno no cliente + medição semanal de adoção Customer Success
4 Violação de LGPD (dados pessoais de clientes no pipeline de IA) Baixa Alto Anonimização antes da ingestão + DPO externo + mapeamento de fluxo conforme Art. 7 da LGPD Jurídico / DPO
5 Dependência de API de terceiros (Zendesk/Freshdesk muda termos) Média Médio Camada de abstração de integração + conectores para ao menos 3 plataformas distintas desde a fase 1 Tech Lead
6 Custo de LLM acima do previsto em escala Alta Médio Limites de processamento por cliente no MVP + cache de respostas + avaliação de modelos open-source CTO / Financeiro
7 Concorrente lança produto similar antes do Go-to-Market Baixa Médio Acelerar MVP para máx. 6 semanas + proteger diferencial de UX e integração com plano de ação CEO / PM
8 Perda de membro-chave do time de IA durante o desenvolvimento Baixa Alto Documentação técnica em repositório versionado + conhecimento crítico distribuído entre 2+ devs CTO
